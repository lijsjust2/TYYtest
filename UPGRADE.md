## 🔄 升级步骤

### 从 v1.x 升级到 v2.0.0

#### 1. 备份现有配置

```bash
# 备份环境变量
cp .env .env.backup

# 备份 Token 文件（如果存在）
cp -r .token .token.backup
```

#### 2. 更新代码

```bash
# 拉取最新代码
git pull origin main

# 或者重新克隆
git clone https://github.com/lijsjust2/TYYsign.git
cd TYYsign
```

#### 3. 安装新依赖

```bash
npm install
```

**新增依赖：**
- `cloud189-sdk@1.0.8`

#### 4. 更新环境变量

**新增环境变量（可选）：**

```bash
# Bark 推送（推荐）
BARK_KEY="your_bark_key"
BARK_SERVER="https://api.day.app"
BARK_GROUP="天翼签到"

# 调试模式（可选）
CLOUD189_VERBOSE=0
```

**原有环境变量（无需修改）：**
- `TYYS` - 账号密码配置
- `DINGTALK_TOKEN` - 钉钉推送
- `DINGTALK_SECRET` - 钉钉加签
- 其他推送配置

#### 5. 测试运行

```bash
npm start
```

检查：
- ✅ 签到是否成功
- ✅ 推送是否正常
- ✅ 日志是否正确记录

## 🐛 常见问题

### Q1: 升级后签到失败怎么办？

**A:** 请检查以下几点：
1. 确认依赖是否正确安装：`npm install`
2. 检查环境变量是否正确配置
3. 尝试删除 `.token` 目录，重新登录
4. 开启调试模式：`CLOUD189_VERBOSE=1`

### Q2: 推送未收到怎么办？

**A:** 请检查以下几点：
1. 确认推送环境变量是否配置
2. 检查推送服务的 token/key 是否正确
3. 确认网络连接正常
4. 查看日志，确认推送是否发送

### Q3: 如何使用 Bark 推送？

**A:** 请参考以下步骤：
1. 在 iPhone 上安装 Bark App
2. 打开 App，复制显示的密钥
3. 配置环境变量：
   ```bash
   BARK_KEY="your_bark_key"
   BARK_GROUP="天翼签到"
   ```

### Q4: 日志文件占用空间怎么办？

**A:** 无需担心，日志会自动清理：
- 单个日志文件最大 10MB
- 保留 3 个备份
- 自动压缩旧日志

### Q5: 如何回退到旧版本？

**A:** 如果需要回退：
```bash
# 回退代码
git checkout v1.x.x

# 恢复依赖
npm install

# 恢复配置
cp .env.backup .env
```
